> 日期：2026-03-01
> 状态：已确定

## 背景

Your Podcast 是一个每日中文科技播客自动生成项目。核心流程：RSS 抓取 → Gemini 筛选 → Podcastfy 生成对话脚本 → GLM TTS 合成 → MP3 输出 → Web 播放。

开发者当前在英国，需要选择合适的云平台和架构方案。

## 架构方案对比

### 方案 A：Next.js + FastAPI（原方案）

```
frontend/ (Next.js)  ←→  backend/ (FastAPI)
```

- 优点：前后端分离，前端交互能力强
- 缺点：两个服务维护成本高，MVP 阶段过度工程化，部署复杂

### 方案 B：纯 FastAPI + Jinja2（推荐方案）

```
app/ (FastAPI + Jinja2 模板)
```

- 优点：单服务部署，简单高效，播放器不需要 React
- 缺点：前端交互能力有限（但 MVP 不需要）

### 方案 C：纯静态站 + GitHub Actions

```
GitHub Actions 生成 → 静态 JSON + MP3 → 静态托管
```

- 优点：零运维，完全免费
- 缺点：无法手动触发生成，扩展性差

## 最终决策

### 选择：Railway + Cloudflare R2 + FastAPI (Jinja2)

### 选择理由

**1. Railway 是最佳云平台**

- **长任务无限制**：播客生成流程需要几分钟（RSS + Gemini + TTS + ffmpeg），Railway 容器常驻运行，无超时限制。CloudBase 云函数和 Vercel Serverless 都有超时问题，这是硬伤。
- **英国延迟低**：20-50ms vs CloudBase 的 200-350ms。开发调试和用户访问体验都更好。
- **CLI 体验最好**：`railway up` 一条命令部署，自动识别 Python 项目，体验最接近 CloudBase 的 `tcb deploy`。
- **性价比高**：Hobby 计划 $5/月含 $5 免费额度，基本零成本。

**2. Cloudflare R2 是最佳存储方案**

- 免费 10GB 存储（够存数百期播客）
- 出站流量完全免费（这是关键，MP3 文件传输量大）
- 全球 CDN，英国和国内访问都快
- S3 兼容 API，代码迁移容易

**3. FastAPI + Jinja2 优于 Next.js + FastAPI**

- MVP 阶段播放页面简单，不需要 React 框架
- 单服务部署，运维成本低
- 减少一个服务 = 减少一半部署和调试工作
- 后续如需复杂前端，可以随时迁移到 Next.js

**4. SQLite + Litestream 优于托管数据库**

- 零运维，无额外费用
- Litestream 自动备份到 R2，数据安全有保障
- 播客元信息数据量小，SQLite 完全够用

### 排除理由

| 平台 | 排除原因 |
|------|---------|
| **CloudBase** | 云函数超时限制不适合长任务；英国延迟 200-350ms 太高 |
| **Fly.io** | 新用户无免费额度；配置比 Railway 复杂 |
| **Vercel** | Serverless 超时限制（最长 300s）不适合播客生成任务 |
| **Supabase** | 定位是 BaaS，不适合跑计算密集型后端任务 |
| **Zeabur** | CLI 工具不够成熟；欧洲节点覆盖一般 |

## 最终架构

```
┌─ 定时触发 ──────────────────────────────────────────────────┐
│  GitHub Actions (cron 每天)  或  Railway 内 APScheduler      │
└─────────────────────┬──────────────────────────────────────-┘
                      ▼
┌─ Railway (欧洲节点) ───────────────────────────────────────-┐
│  FastAPI                                                     │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  ┌──────────┐   │
│  │ RSS 抓取  │→│ Gemini   │→│ Podcastfy  │→│ GLM TTS  │   │
│  │feedparser │  │ 筛选8-10 │  │ 对话脚本   │  │ 逐句合成  │   │
│  └──────────┘  └──────────┘  └───────────┘  └────┬─────┘   │
│                                                    │         │
│  ┌──────────┐                    ┌────────────────┘         │
│  │ SQLite   │← 存元信息          ▼                          │
│  │ Litestream│                 ffmpeg 合并 MP3               │
│  │  备份到R2 │                    │                          │
│  └──────────┘                    ▼                          │
│                           上传 MP3 到 R2                     │
└──────────────────────────────────────────────────────────---┘
                      ▼
┌─ Cloudflare R2 (全球 CDN) ─────────────────────────────────┐
│  /episodes/2026-03-01.mp3                                    │
│  免费 10GB 存储 + 免费出站流量                                 │
└─────────────────────┬──────────────────────────────────────-┘
                      ▼
┌─ 前端 (FastAPI Jinja2 同服务) ─────────────────────────────-┐
│  - 播客播放器页面                                             │
│  - 往期列表页面                                               │
│  - MP3 链接指向 R2 CDN                                       │
└────────────────────────────────────────────────────────────-┘
```

## 技术栈总结

| 层 | 选型 | 说明 |
|----|------|------|
| 云平台 | Railway | CLI 部署，欧洲节点，容器无超时 |
| 后端 | FastAPI + Jinja2 | 单服务搞定 API + 页面渲染 |
| RSS | feedparser | 成熟稳定的 RSS 解析库 |
| AI 筛选 | Google Gemini | 从全部文章挑 8-10 篇最有价值的 |
| 脚本生成 | Podcastfy（备选 Gemini prompt） | 生成双主持人对话脚本 |
| TTS | GLM 智谱 | 中文效果好，支持双声线 |
| 音频处理 | ffmpeg (pydub) | 拼接语音片段为 MP3 |
| 文件存储 | Cloudflare R2 | 免费 10GB，全球 CDN，S3 兼容 |
| 数据库 | SQLite + Litestream | 零运维，自动备份到 R2 |
| 定时任务 | GitHub Actions cron | 免费 2000 分钟/月 |
